# 安全策略

## 报告漏洞

如果您发现安全漏洞，请通过以下方式负责任地披露：

- 发送邮件至：idcu@qq.com
- 邮件标题前缀：[Lyt.js Security]

我们会在 48 小时内确认收到，并在 7 个工作日内回复处理计划。

## 支持的版本

| 版本 | 支持状态 |
|------|---------|
| v4.x | ✅ 积极支持 |
| v1.x | ✅ 积极支持 |
| v0.x | ❌ 不再支持 |

## 安全更新策略

- **积极支持的版本**（v4.x、v1.x）会在发现安全漏洞后尽快发布补丁版本
- 安全补丁会通过 GitHub Release 和 CHANGELOG.md 进行公告
- 建议用户始终升级到最新的补丁版本以获取安全修复
- 当一个主版本进入不再支持状态时，将不再接收安全更新

## CVE 披露流程

1. **发现漏洞**：安全研究者通过上述联系方式提交漏洞报告
2. **确认与评估**：维护团队在 48 小时内确认收到，评估漏洞严重程度
3. **修复开发**：根据严重程度，在合理时间内完成修复（通常 7 个工作日内）
4. **协调披露**：在补丁发布前，与报告者协调披露时间线
5. **发布补丁**：通过 GitHub Release 发布包含修复的补丁版本
6. **公开公告**：在补丁发布后，通过 CHANGELOG.md 和 GitHub Advisory 公开漏洞详情
