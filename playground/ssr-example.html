StringRenderer

0 概览

Lyt.js 的 SSR 方案基于 StringRenderer，将 VNode 树同步序列化为 HTML 字符串。 无需真实 DOM 环境，适用于 Node.js 服务端渲染场景。

renderToString

将 VNode 树同步渲染为完整 HTML 字符串

HTML 转义

自动转义特殊字符，防止 XSS 攻击

Fragment 支持

多根节点组件无需包裹元素

属性序列化

支持 class/style/布尔属性等多种格式

1 StringRenderer 基本用法

StringRenderer 是 SSR 的核心类，提供 createElement、createText、insert 和 renderToString 方法。 在服务端环境中，没有真实 DOM，所有操作都基于轻量描述对象。

server.js 
import { StringRenderer } from '@lyt/renderer'

// 创建渲染器实例
const renderer = new StringRenderer()

// 创建元素描述对象（非真实 DOM）
const parent = renderer.createElement('div')
const text = renderer.createText('Hello SSR')

// 组装节点树
renderer.insert(parent, text)
// parent.children = [{ type: 'text', value: 'Hello SSR' }]

2 renderToString 渲染

renderToString 接收一个 VNode，递归遍历整棵树，输出完整的 HTML 字符串。 支持嵌套元素、文本内容、属性序列化、自闭合标签和 Fragment。

render.js 
// 简单元素
const vnode1 = { type: 'div', props: null, children: 'Hello', shapeFlag: 9 }
renderer.renderToString(vnode1)
// => '<div>Hello</div>'

// 嵌套元素
const vnode2 = {
  type: 'ul', props: { class: 'list' },
  children: [
    { type: 'li', props: null, children: 'Item 1', shapeFlag: 9 },
    { type: 'li', props: null, children: 'Item 2', shapeFlag: 9 },
  ],
  shapeFlag: 17
}
renderer.renderToString(vnode2)
// => '<ul class="list"><li>Item 1</li><li>Item 2</li></ul>'

// 自闭合标签
const vnode3 = { type: 'img', props: { src: '/logo.png' }, shapeFlag: 1 }
renderer.renderToString(vnode3)
// => '<img src="/logo.png" />'

3 HTML 转义（防 XSS）

StringRenderer 自动对文本内容和属性值进行 HTML 实体转义， 防止用户输入的内容被当作 HTML 解析，确保安全性。

escape.js 
// 用户输入包含恶意脚本
const userInput = '<script>alert("xss")</script>'

const vnode = {
  type: 'div', props: null,
  children: userInput,
  shapeFlag: 9
}

renderer.renderToString(vnode)
// => '<div>&lt;script&gt;alert(&quot;xss&quot;)&lt;/script&gt;</div>'
// 浏览器会显示纯文本，不会执行脚本
