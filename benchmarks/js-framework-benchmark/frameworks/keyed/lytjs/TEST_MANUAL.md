# LytJS 性能测试手册

本手册用于指导如何在 js-framework-benchmark 中进行 LytJS 的性能测试。

## 💻 推荐测试环境

为了获得准确且可复现的测试结果，建议使用以下环境：

### 硬件配置

| 组件 | 最低配置 | 推荐配置 | 理想配置 |
|------|---------|---------|---------|
| CPU | Intel i5 / AMD Ryzen 5 | Intel i7-12700H / AMD Ryzen 7 5800H | Intel i9-13900K / AMD Ryzen 9 7950X |
| 内存 | 8 GB | 16 GB | 32 GB+ |
| 存储 | SSD | NVMe SSD | NVMe SSD |
| 电源 | 连接电源（笔记本） | 连接电源 | 连接电源 |

### 软件配置

| 组件 | 推荐 | 说明 |
|------|------|------|
| 操作系统 | Windows 11 / macOS 13+ / Linux (Ubuntu 22.04+) | 使用最新稳定版 |
| 浏览器 | Chrome 130+ / Edge 130+ | 与官方测试环境一致（Chrome 148） |
| 浏览器模式 | 无痕模式 | 避免扩展和缓存干扰 |
| 扩展程序 | 全部禁用 | 避免影响性能测量 |
| 电源计划 | 高性能 / 平衡 | 确保 CPU 不降频 |

### 测试前准备

1. 关闭所有其他应用程序
2. 确保没有后台更新或扫描任务
3. 连接电源（笔记本用户）
4. 设置电源计划为高性能
5. 关闭浏览器自动更新
6. 等待系统完全稳定后开始测试

## 🚀 测试步骤

### 快速开始

1. 启动本地服务器：
```bash
cd benchmarks/js-framework-benchmark/frameworks/keyed/lytjs
python -m http.server 8080
```

2. 在浏览器中打开：http://localhost:8080/compare.html

3. 点击「测量全部三种模式」开始测试

### 详细测试

#### 1️⃣ 基础功能验证

首先验证所有操作是否正常工作：
- 创建 1000 行 - 点击按钮，表格应显示 1000 行数据
- 更新 10% - 每 10 行的标签应加上 "!!!"
- 选择行 - 点击某行，应高亮显示
- 交换行 - 第 2 行和第 999 行应互换位置
- 删除行 - 点击删除按钮，对应行应消失
- 追加 1000 行 - 表格应从 1000 行增加到 2000 行
- 清空 - 表格应清空

#### 2️⃣ 性能测试模式

打开 [compare.html](./compare.html) 页面：

**模式选择：**
- VDOM 模式 - 传统虚拟 DOM 实现
- Vapor 模式 - 编译优化的无虚拟 DOM 实现（默认，官方提交）
- Signal 模式 - 细粒度响应式信号实现

**测量方法：**
1. 确保系统处于空闲状态
2. 点击「测量当前模式」只测量选中的模式
3. 点击「测量全部三种模式」依次测量所有模式
4. 等待所有测试完成（约 30-40 秒）
5. 查看结果表格

#### 3️⃣ 结果记录

测试完成后，建议记录以下信息：
- 测试环境（CPU、内存、操作系统、浏览器版本）
- 三种模式的平均耗时
- 与官方数据的对比情况
- 是否有异常波动

## 🔍 常见问题

### Q: 测试结果波动很大怎么办？

A:
1. 确保没有其他程序占用 CPU
2. 多次测试取平均值
3. 使用无痕模式，关闭所有扩展
4. 笔记本确保连接电源

### Q: 与官方数据有差异正常吗？

A:
完全正常。不同硬件配置、浏览器版本、操作系统都会导致结果差异。
- 差异在 ±30% 内是可接受的
- 主要关注相对性能，而非绝对数值
- 在同一台机器上对比不同框架更有意义

### Q: 如何复现官方测试结果？

A:
1. 使用相同硬件环境（参考官方基准数据）
2. 使用相同版本的 Chrome 浏览器（Chrome 148）
3. 严格按照官方测试流程
4. 参考官方提供的测试脚本
