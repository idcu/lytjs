# 生态与社区

LytJS 致力于打造一个健康、活跃的开源生态系统。我们欢迎并鼓励社区成员贡献代码、分享组件和插件，共同推动 LytJS 的发展。

## 生态组成

### 核心生态

LytJS 官方提供了完整的核心生态系统：

- **响应式系统**：`@lytjs/reactivity`
- **虚拟 DOM**：`@lytjs/vdom`
- **编译器**：`@lytjs/compiler`
- **核心框架**：`@lytjs/core`
- **UI 组件库**：`@lytjs/ui`
- **路由**：`@lytjs/router`
- **状态管理**：`@lytjs/store`
- **SSR**：`@lytjs/ssr`
- **DevTools**：`@lytjs/devtools`

### 官方插件

官方插件经过严格测试和优化：

- **主题插件**：`@lytjs/plugin-theme`
- **日志插件**：`@lytjs/plugin-logger`
- **国际化**：`@lytjs/plugin-i18n`
- **认证插件**：`@lytjs/plugin-auth`
- **存储插件**：`@lytjs/plugin-storage`
- **图表插件**：`@lytjs/plugin-chart`

## 社区贡献

### 贡献类型

我们欢迎各种形式的贡献：

#### 1. 代码贡献

- 修复 Bug
- 实现新功能
- 优化性能
- 改进代码质量

#### 2. 文档贡献

- 改进现有文档
- 翻译文档
- 编写教程
- 添加代码示例

#### 3. 社区贡献

- 回答问题
- 分享经验
- 组织活动
- 推广 LytJS

### 如何贡献代码

1. **Fork 项目**：Fork `https://gitee.com/lytjs/lytjs`
2. **创建分支**：`git checkout -b feature/your-feature`
3. **开发开发**：按照 [开发文档](../development/) 进行开发
4. **提交代码**：`git commit -m "feat: add new feature"`
5. **推送代码**：`git push origin feature/your-feature`
6. **创建 PR**：在 Gitee 上创建 Pull Request

### 代码规范

贡献代码时请遵循以下规范：

- 遵循项目的 [编码规范](../guide/typescript)
- 确保通过所有测试
- 添加适当的测试用例
- 更新相关文档

## 第三方生态

### 组件库

我们鼓励社区开发高质量的组件库：

#### 优秀组件库标准

1. **代码质量**
   - 完整的 TypeScript 类型支持
   - 遵循零依赖原则
   - 通过单元测试

2. **文档质量**
   - 完整的使用文档
   - 丰富的代码示例
   - API 文档清晰

3. **维护更新**
   - 及时跟进框架版本
   - 积极响应 Issue
   - 定期更新维护

### 插件开发

如果你是插件开发者：

1. 参考 [插件开发指南](../development/PLUGIN_DEVELOPMENT.md)
2. 使用 `definePlugin` API
3. 确保零第三方依赖
4. 提供完整测试

### 包发布

建议的包命名规范：

```
@lytjs-community/{plugin-name}    # 社区插件
@lytjs-community/{component-name} # 社区组件
@{username}/lytjs-{name}           # 个人发布的包
```
