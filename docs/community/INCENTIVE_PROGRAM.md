# LytJS 社区激励计划

> 版本: v1.0.0
> 创建日期: 2026-05-16
> 状态: 草案

---

## 一、计划愿景

LytJS 社区激励计划旨在鼓励开发者积极参与生态系统建设，通过代码贡献、文档完善、问题解答等方式，共同推动 LytJS 的发展。

---

## 二、激励方式

### 2.1 代码贡献激励

| 贡献类型 | 奖励内容 | 认定标准 |
|----------|----------|----------|
| 核心功能 PR | LytJS 官方周边 + 特别感谢徽章 | PR 被合并且功能完整 |
| Bug 修复 PR | LytJS 贴纸套装 | PR 被合并 |
| 性能优化 PR | 特别感谢徽章 | 性能提升 > 10% |
| 插件开发 | 官方推荐 + 文档展示 | 通过审核标准 |

### 2.2 文档贡献激励

| 贡献类型 | 奖励内容 | 认定标准 |
|----------|----------|----------|
| 教程文章 | 官方博客展示 + 社交媒体推广 | 被采纳并发布 |
| 翻译贡献 | 特别贡献者徽章 | 完整翻译某个模块 |
| 案例分享 | 官方示例展示 | 被收录到示例库 |
| 文档纠错 | LytJS 贴纸 | 有效纠错 10+ 处 |

### 2.3 社区贡献激励

| 贡献类型 | 奖励内容 | 认定标准 |
|----------|----------|----------|
| Issue 回答 | 社区贡献者徽章 | 回答被标记为有用 |
| 问题解决 | 月度活跃贡献者称号 | 月度帮助解决问题 5+ |
| 代码审查 | 审查贡献者徽章 | 审查 PR 5+ |
| 社区活动 | 免费参与线下活动 | 活动组织/主持 |

---

## 三、贡献等级

### 3.1 等级划分

| 等级 | 称号 | 要求 | 专属权益 |
|------|------|------|----------|
| 🥉 铜牌 | 社区贡献者 | 贡献积分 100+ | 社区徽章 |
| 🥈 银牌 | 活跃贡献者 | 贡献积分 500+ | 铜牌权益 + 官方感谢 |
| 🥇 金牌 | 核心贡献者 | 贡献积分 1000+ | 银牌权益 + 预览功能 |
| 💎 钻石 | 社区大使 | 贡献积分 5000+ | 金牌权益 + 核心团队接触 |

### 3.2 贡献积分规则

| 贡献类型 | 积分 |
|----------|------|
| 核心代码 PR 合并 | 100 |
| 插件/工具 PR 合并 | 50 |
| Bug 修复 PR 合并 | 30 |
| 文档 PR 合并 | 20 |
| Issue 有效回答 | 5 |
| Issue 提交（有效） | 3 |
| 代码审查参与 | 5 |
| 社区活动参与 | 10 |

## 四、季度评选

### 4.1 季度之星

每季度评选 3 名季度之星：

- **创新之星**：提出并实现创新功能
- **守护之星**：修复关键 Bug，保障稳定性
- **布道之星**：推广 LytJS，培训新手

### 4.2 年度盛典

每年年底举办年度盛典，评选：

- 年度贡献者 TOP 10
- 最佳新贡献者
- 最佳社区经理
- 特别贡献奖

---

## 五、特别计划

### 5.1 Hacktoberfest 参与

LytJS 积极参与 Hacktoberfest：

- 标记 beginner-friendly 的 Issue
- 提供详细的任务指南
- 及时审查和合并 PR
- 发放 Hacktoberfest 官方徽章

### 5.2 开源之夏

参与开源之夏/Google Summer of Code：

- 发布 Ideas List
- 提供导师指导
- 设立专项奖学金
- 优先考虑核心功能

---

## 六、获取方式

### 6.1 自动获取

以下激励会自动发放：

- 贡献积分：GitHub 自动统计
- 徽章：在贡献者页面自动展示
- 特别感谢：在 CHANGELOG 中自动添加

### 6.2 申请获取

以下激励需要申请：

1. **周边奖品**：填写申请表 + 贡献证明
2. **线下活动资格**：提前报名 + 贡献审核
3. **高级称号**：达到积分阈值后自动晋升

---

## 七、申请流程

### 7.1 周边奖品申请

1. 在 GitHub Issue 提交申请
2. 提供贡献证明链接
3. 填写收货地址（如适用）
4. 等待审核（3 工作日）
5. 发放奖励

### 7.2 称号晋升

称号晋升自动进行，无需申请。当贡献积分达到阈值后，系统会自动更新称号。

---

## 八、注意事项

### 8.1 作弊行为

以下行为会导致激励被收回：

- 刷积分行为
- 提交无意义 PR
- 抄袭他人代码
- 违反社区行为准则

### 8.2 版权说明

- 所有贡献代码遵循 MIT 许可证
- 贡献者保留代码署名权
- 官方有权使用贡献内容进行推广
