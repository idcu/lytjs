# 模块组装说明

这篇文档详细说明 Lyt.js 各个模块如何协同工作，以及如何把它们组装在一起。

## 🏗️ 整体架构概览

首先看一下各个模块的依赖关系：

```
┌─────────────────────────────────────────┐
│         lytjs (聚合包)                   │
│  整合所有模块，提供统一入口                │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│         core (核心入口)                  │
│  createApp, 插件系统, Web Component      │
└─────────────────────────────────────────┘
         ↓                    ↓
┌──────────────┐    ┌─────────────────┐
│  renderer    │    │   component     │
│  渲染器      │    │  组件系统       │
└──────────────┘    └─────────────────┘
         ↓                    ↓
┌─────────────────────────────────────────┐
│       reactivity (响应式系统)            │
│  所有功能模块都依赖这个基础                │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│     compiler (模板编译器)                │
└─────────────────────────────────────────┘

其他功能模块 (可选):
┌─────────┐ ┌─────────┐ ┌───────┐
│ router  │ │  store  │ │  cli  │
│ 路由    │ │ 状态管理 │ │ 工具  │
└─────────┘ └─────────┘ └───────┘
```

## 🔧 核心流程：从模板到页面

让我们看看一个完整的应用是如何工作的：

### 1. 创建应用

```typescript
import { createApp } from '@lytjs/core'

const app = createApp({
  template: `<div>{{ message }}</div>`,
  data() {
    return {
      message: 'Hello'
    }
  }
})

app.mount('#app')
```

### 2. 内部发生了什么

```
┌─────────────────────────────────────────┐
│  1. createApp 被调用                     │
│     ├─ 创建应用实例                      │
│     └─ 处理根组件选项                    │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  2. 编译模板 (compiler)                  │
│     ├─ parse: 生成 AST                  │
│     ├─ transform: 处理指令               │
│     └─ codegen: 生成渲染函数             │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  3. 组件实例化 (component)               │
│     ├─ 创建组件实例                      │
│     ├─ 初始化 props                     │
│     └─ 初始化 data (响应式)              │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  4. 响应式绑定 (reactivity)              │
│     └─ data 被 reactive() 包装           │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  5. 初始渲染 (renderer)                  │
│     ├─ 执行渲染函数 → VNode              │
│     └─ 挂载 VNode → DOM                  │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  6. 响应式更新                          │
│     ├─ 数据变化                          │
│     ├─ 触发 effect                       │
│     ├─ 重新执行渲染函数                  │
│     └─ patch DOM                         │
└─────────────────────────────────────────┘
```

## 📦 详细模块组装说明

让我们看看每个模块是如何被组装在一起的。

### 1. Reactivity：基础中的基础

Reactivity 是唯一被所有其他模块依赖的模块。

**关键点**：
- 所有模块都可以独立使用 reactivity
- reactivity 不依赖其他任何模块

### 2. Compiler：独立工具

Compiler 是一个独立的工具，可以单独使用。

**关键点**：
- Compiler 可以独立运行
- Compiler 不依赖 reactivity 或 renderer
- 生成的代码是通用的，不绑定特定实现

### 3. Renderer：渲染抽象

Renderer 是一个可配置的渲染器，可以搭配不同的后端。

**关键点**：
- Renderer 可以独立使用
- Renderer 可以不依赖 Component
- Renderer 只需要知道如何操作目标平台

### 4. Component：使用 Reactivity 和 Renderer

Component 模块是第一个把多个模块组装在一起的。

**关键点**：
- Component 依赖 reactivity 和 renderer
- Component 不依赖 compiler（渲染函数可以手写）
- Component 可以独立使用

### 5. Core：整合所有核心模块

Core 模块整合了前面的所有模块。

### 6. lytjs：聚合包

lytjs 模块只是重新导出所有模块，不包含任何实现。

## 🎯 模块设计原则

Lyt.js 的模块设计遵循以下原则：

### 1. 单向依赖

```
reactivity ← compiler
            ← renderer
            ← component
            ← router
            ← store

component ← core
renderer  ← core
compiler  ← core

core ← lytjs
```

### 2. 低耦合，高内聚

每个模块都有明确的职责，不互相依赖：

- **reactivity**：只负责响应式
- **compiler**：只负责编译模板
- **renderer**：只负责渲染 VNode
- **component**：只负责组件逻辑
- **core**：只负责整合

### 3. 可独立使用

每个模块都可以单独使用，不需要引入整个框架。

## 🔬 数据流追踪

让我们追踪一个完整的数据变化流程：

```
用户交互
  ↓
DOM 事件触发
  ↓
执行事件处理器
  ↓
修改数据 (state.count++)
  ↓
Proxy set 拦截
  ↓
trigger() 触发
  ↓
找到对应的 effect
  ↓
执行 effect 回调
  ↓
调用 render() 重新渲染
  ↓
生成新的 VNode 树
  ↓
patch() 对比新旧 VNode
  ↓
更新真实 DOM
  ↓
用户看到变化
```
